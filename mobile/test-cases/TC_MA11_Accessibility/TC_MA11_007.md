## Scenario
The browser's Screen Orientation API must report the correct orientation type when the viewport is in portrait or landscape mode.

## Preconditions
- MPC_001 — Mobile browser open, network available
- MPC_002 — CardioCheck app accessible at https://cardiocheck-releasev140.up.railway.app

## Test Data
| Field    | Value                          |
|----------|--------------------------------|
| Device   | Pixel 5 (393×851, Android emu) |

## Steps
1. Set viewport to 393×851 (portrait) on Pixel 5 emulation
2. Navigate to https://cardiocheck-releasev140.up.railway.app
3. Execute `screen.orientation.type` in the browser console
4. Verify the value is `'portrait-primary'` or `'portrait-secondary'`
5. Switch viewport to 851×393 (landscape)
6. Execute `screen.orientation.type` again
7. Verify the value is `'landscape-primary'` or `'landscape-secondary'`

## Expected Result
`screen.orientation.type` returns the correct orientation type for both portrait and landscape viewport configurations.
