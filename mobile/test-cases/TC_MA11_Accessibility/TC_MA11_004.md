## Scenario
All interactive elements (buttons, links, inputs) must meet the minimum 44×44 CSS pixel touch target size per WCAG 2.5.5 Success Criterion.

## Preconditions
- MPC_001 — Mobile browser open, network available
- MPC_002 — CardioCheck app accessible at https://cardiocheck-releasev140.up.railway.app
- MPC_003 — User is logged in

## Test Data
| Field       | Value                          |
|-------------|--------------------------------|
| Minimum     | 44×44 CSS pixels               |
| Device      | Pixel 5 (393×851, Android emu) |

## Steps
1. Navigate through login, dashboard, and patient form pages
2. On each page, query all interactive elements: `document.querySelectorAll('button, a, input, select, [role="button"], [tabindex]')`
3. For each element, retrieve `getBoundingClientRect()` and check width ≥ 44 and height ≥ 44
4. Log all elements that fail the size check with their selector and actual dimensions

## Expected Result
All primary interactive elements have touch targets ≥ 44×44 CSS pixels. Any elements below the threshold are flagged with their selector, page, and actual dimensions.
