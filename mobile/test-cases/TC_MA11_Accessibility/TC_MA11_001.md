The login page must be audited with axe-core for WCAG 2.1 AA compliance; all violations must be recorded for remediation tracking.

- MPC_001 — Mobile browser open, network available
- MPC_002 — CardioCheck app accessible at https://cardiocheck-releasev140.up.railway.app

## Test Data
| Field    | Value                          |
|----------|--------------------------------|
| Tool     | axe-core via @axe-core/playwright |
| Standard | WCAG 2.1 Level AA              |
| Device   | Pixel 5 (393×851, Android emu) |

## Steps
1. Navigate to https://cardiocheck-releasev140.up.railway.app on Pixel 5 emulation
2. Confirm the login page is loaded
3. Inject and run `axe.run()` via Playwright `page.evaluate()`
4. Collect all violations from the audit result
5. Log each violation with: description, impact level, HTML element, and help URL

The axe audit runs without errors. All violations are captured and logged. Zero Critical or Serious violations is the target. Any violations found are documented for the bug report.
