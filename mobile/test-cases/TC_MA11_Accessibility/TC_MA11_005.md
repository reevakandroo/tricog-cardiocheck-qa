## Scenario
All form inputs (login, patient form) must have associated accessible labels via `<label>`, `aria-label`, or `aria-labelledby` so screen readers can identify each field.

## Preconditions
- MPC_001 — Mobile browser open, network available
- MPC_002 — CardioCheck app accessible at https://cardiocheck-releasev140.up.railway.app

## Test Data
| Field    | Value                          |
|----------|--------------------------------|
| Device   | Pixel 5 (393×851, Android emu) |

## Steps
1. Navigate to https://cardiocheck-releasev140.up.railway.app on Pixel 5 emulation
2. On the login page, inspect each input element for: `aria-label`, `aria-labelledby`, or an associated `<label>` with matching `for` attribute
3. Navigate to the patient form and repeat the inspection
4. Log any inputs that lack accessible labels

## Expected Result
All form inputs have accessible labels via `aria-label`, `aria-labelledby`, or associated `<label>` elements. No unlabeled inputs exist. Screen reader users can identify all fields.
