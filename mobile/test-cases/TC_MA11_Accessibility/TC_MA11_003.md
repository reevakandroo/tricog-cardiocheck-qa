## Scenario
When the user increases browser font size to 200%, the login and dashboard pages must remain usable without text overflow, overlapping elements, or hidden content.

## Preconditions
- MPC_001 — Mobile browser open, network available
- MPC_002 — CardioCheck app accessible at https://cardiocheck-releasev140.up.railway.app

## Test Data
| Field      | Value                          |
|------------|--------------------------------|
| Font Scale | 200% (browser zoom / CSS font-size override) |
| Device     | Pixel 5 (393×851, Android emu) |

## Steps
1. Navigate to https://cardiocheck-releasev140.up.railway.app on Pixel 5 emulation
2. Inject CSS to set `html { font-size: 200%; }` or use `page.evaluate(() => document.documentElement.style.fontSize = '200%')`
3. Observe the login page layout
4. Log in and observe the dashboard layout
5. Check for text overflow, overlapping elements, or hidden buttons

## Expected Result
At 200% font size, all text is readable and no elements overlap or become inaccessible. Content may require vertical scrolling but no horizontal overflow should occur. Form fields and buttons remain functional.
