## Scenario
When navigating the login and dashboard pages using a keyboard (Tab key), each focused element must display a visible focus ring/outline so keyboard-only users can track their position.

## Preconditions
- MPC_001 — Mobile browser open, network available
- MPC_002 — CardioCheck app accessible at https://cardiocheck-releasev140.up.railway.app

## Test Data
| Field    | Value                          |
|----------|--------------------------------|
| Device   | Pixel 5 (393×851, Android emu) |
| Input    | Keyboard Tab navigation        |

## Steps
1. Navigate to https://cardiocheck-releasev140.up.railway.app on Pixel 5 emulation
2. Press Tab key to focus the first interactive element
3. Verify a visible focus ring/outline is displayed on the focused element
4. Press Tab again to move to the next element
5. Verify focus ring follows and is visible
6. Continue tabbing through all interactive elements on the login page
7. Log any elements where the focus ring is absent or invisible (outline: none without alternative)

## Expected Result
All interactive elements display a visible focus ring when focused. No element has `outline: none` without an alternative focus indicator. The focus order is logical (top-to-bottom, left-to-right).
