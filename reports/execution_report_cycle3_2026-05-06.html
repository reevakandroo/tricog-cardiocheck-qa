⚠️ Critical App Bug: Session Expiry Mid-Workflow (Affects 6 Tests)
Root Cause: The CardioCheck app session expires in approximately 3–5 minutes. Long test workflows (ECG seed → patient form fill → risk assessment API call) regularly exceed this window, causing the app to silently redirect to /login mid-flow. When the session is lost before the risk result is rendered, the export icon never appears and these tests skip their precondition guard (if exportBtn.count === 0 → test.skip).

Affected Tests: TC_RSK_002 (Moderate risk — Fail), TC_RPT_002, TC_RPT_003, TC_RPT_008, TC_RPT_009, TC_RPT_011 (Skipped)

Impact: Real-world users performing the same flow (open ECG → fill patient data → await risk result) would be silently logged out mid-session, losing their work with no warning — a Critical UX and data safety issue.

Recommended Fix: Increase session TTL to ≥ 30 minutes, implement silent token refresh on active use, and add an explicit session-expiry warning dialog before auto-logout.
